# Skill: Search Knowledge Hub

用于在 Knowledge Hub 中渐进式检索相关经验，避免一次性拉取过多无用文本。

## 步骤

1. **提取关键词**：从当前任务中识别核心关键词（如 `k8s`、`timeout`、`502`）和可能的 tags。

2. **浏览目录**：调用 `kh_browse`（不传参数）查看知识库顶层 tag 分布和覆盖范围。
   - 若已知相关 tag，传入 `selected_tags` 进行下钻，缩小范围。
   - 重复下钻直到 `total_matches` ≤ 10 或找到明确方向。

3. **搜索知识**：调用 `kh_search`，传入积累的 tags 和关键词，获取标题 + 摘要列表（按权重排序）。

4. **筛选相关条目**：阅读摘要，挑选 1-2 条最相关的，不要贪多（Token 有限）。

5. **读取全文**：对选中的条目调用 `kh_read_full` 获取完整内容和评论概要。

6. **整合应用**：将获取的知识整合到当前任务计划中，并在 `memory/scratchpad.md` 记录使用了哪些知识。

## 注意事项

- 若 `kh_browse` 返回的 tag 与当前任务完全不相关，直接跳到 `kh_search` 用关键词搜索。
- 若搜索无结果，说明知识库尚无相关内容，自行解决后通过 `/reflect` 贡献经验。
- `kh_read_full` 会更新访问计数，请避免重复调用同一条目。
